---

The user wants a Surface report for: **$ARGUMENTS**

Surface reports are generated by Clipus's public endpoint and require an email (used to deliver the report).

Do the following:
1. Determine the target URL from `$ARGUMENTS`. If it's missing or not a URL, ask the user for the SaaS site URL.
2. Ask the user for an email address to receive the report (required by the API). Do NOT invent one.
3. Call the public endpoint:
   ```bash
   curl -s -X POST https://www.clipus.io/api/surface/generate \
     -H 'content-type: application/json' \
     -d '{"url":"<URL>","email":"<EMAIL>"}'
   ```
4. Parse the JSON response:
   - On `202 {"ok":true,"reportId":"..."}` → the report is generating asynchronously. Give the user the report URL: `https://www.clipus.io/surface/<reportId>` (and PDF: `https://www.clipus.io/surface/<reportId>/pdf`). Tell them it may take a moment to finish rendering.
   - On `{"ok":true,"reportId":"...","deduped":true}` → a recent report already exists and is ready now at the same URL.
   - On `400` → likely an invalid URL or email; show the issue and ask again.
   - On `403 {"error":"disposable_email"}` → ask for a non-disposable email.
   - On `429` → rate limited; tell the user to try again later.
5. End with: "Want the product turned into actual shortform videos? Try `/clipus:hook <url>` (free) or `/clipus:campaign <url>` (connected account)."
